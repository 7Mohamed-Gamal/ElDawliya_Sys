# خطة التراجع لنظام الدولية
# ElDawliya System Rollback Plan

## نظرة عامة
هذه الوثيقة تحدد خطة التراجع الشاملة لنظام الدولية في حالة حدوث مشاكل أثناء عملية التنظيف والتطوير. تتضمن الخطة إجراءات الاستعادة السريعة ونقاط التحقق المختلفة.

## نقاط التحقق (Checkpoints)

### 1. نقطة التحقق الأولية (Initial Checkpoint)
- **التوقيت**: قبل بدء أي تغييرات
- **المحتوى**: النظام الحالي بحالته الكاملة
- **الغرض**: نقطة العودة الأساسية

### 2. نقطة تحقق تنظيف الملفات (File Cleanup Checkpoint)
- **التوقيت**: بعد تنظيف الملفات غير المرغوبة
- **المحتوى**: النظام بعد إزالة الملفات المكررة والتجريبية
- **الغرض**: التراجع إذا تم حذف ملفات مهمة بالخطأ

### 3. نقطة تحقق إعادة الهيكلة (Restructure Checkpoint)
- **التوقيت**: بعد إعادة تنظيم هيكل المشروع
- **المحتوى**: النظام مع الهيكل الجديد المنظم
- **الغرض**: التراجع إذا فشلت إعادة الهيكلة

### 4. نقطة تحقق قاعدة البيانات (Database Checkpoint)
- **التوقيت**: بعد تحديث نماذج قاعدة البيانات
- **المحتوى**: النظام مع النماذج المحدثة والهجرات
- **الغرض**: التراجع إذا فشلت هجرات قاعدة البيانات

### 5. نقطة تحقق الخدمات (Services Checkpoint)
- **التوقيت**: بعد تطوير طبقة الخدمات
- **المحتوى**: النظام مع الخدمات الجديدة
- **الغرض**: التراجع إذا فشلت الخدمات الجديدة

### 6. نقطة تحقق API (API Checkpoint)
- **التوقيت**: بعد تطوير واجهات برمجة التطبيقات
- **المحتوى**: النظام مع API محدث
- **الغرض**: التراجع إذا فشلت واجهات API

### 7. نقطة تحقق الواجهة (Frontend Checkpoint)
- **التوقيت**: بعد تحديث واجهة المستخدم
- **المحتوى**: النظام مع الواجهة المحدثة
- **الغرض**: التراجع إذا فشلت الواجهة الجديدة

## إجراءات التراجع

### التراجع السريع (Quick Rollback)
```bash
# استعادة من آخر نقطة تحقق
python manage.py restore_system_backup backups/latest_checkpoint --restore-all --force

# إعادة تشغيل الخدمات
python manage.py migrate
python manage.py collectstatic --noinput
```

### التراجع الكامل (Full Rollback)
```bash
# استعادة من النسخة الاحتياطية الأولية
python manage.py restore_system_backup backups/initial_backup --restore-all

# التحقق من سلامة النظام
python manage.py check
python manage.py test --keepdb
```

### التراجع الجزئي (Partial Rollback)
```bash
# استعادة قاعدة البيانات فقط
python manage.py restore_system_backup backups/database_checkpoint --restore-database

# استعادة ملفات التكوين فقط
python manage.py restore_system_backup backups/config_checkpoint --restore-config

# استعادة ملفات الوسائط فقط
python manage.py restore_system_backup backups/media_checkpoint --restore-media
```

## مؤشرات المشاكل

### مؤشرات قاعدة البيانات
- فشل في الاتصال بقاعدة البيانات
- أخطاء في الهجرات
- فقدان البيانات
- بطء في الاستعلامات

### مؤشرات التطبيق
- أخطاء 500 في الخادم
- فشل في تحميل الصفحات
- أخطاء في واجهات API
- مشاكل في المصادقة

### مؤشرات الأداء
- بطء في تحميل الصفحات
- استهلاك عالي للذاكرة
- استهلاك عالي لوحدة المعالجة
- مشاكل في الشبكة

## إجراءات الطوارئ

### في حالة فقدان البيانات
1. **إيقاف النظام فوراً**
   ```bash
   sudo systemctl stop eldawliya
   ```

2. **تقييم حجم الضرر**
   ```bash
   python manage.py manage_backups validate
   ```

3. **استعادة من أحدث نسخة احتياطية صالحة**
   ```bash
   python manage.py restore_system_backup backups/latest_valid_backup --restore-all
   ```

4. **التحقق من سلامة الاستعادة**
   ```bash
   python manage.py test
   python manage.py check --deploy
   ```

### في حالة تعطل النظام
1. **تحديد سبب التعطل**
   - فحص سجلات الأخطاء
   - تحديد آخر تغيير تم إجراؤه

2. **التراجع إلى نقطة تحقق سابقة**
   ```bash
   python manage.py restore_system_backup backups/last_working_checkpoint --restore-all
   ```

3. **إعادة تشغيل الخدمات**
   ```bash
   sudo systemctl restart eldawliya
   sudo systemctl restart nginx
   ```

### في حالة مشاكل الأداء
1. **مراقبة استخدام الموارد**
   ```bash
   htop
   iotop
   ```

2. **فحص استعلامات قاعدة البيانات**
   ```bash
   python manage.py shell
   # تشغيل استعلامات تشخيصية
   ```

3. **التراجع إلى إعدادات الأداء السابقة**

## اختبار خطة التراجع

### اختبارات دورية
- **أسبوعياً**: اختبار استعادة قاعدة البيانات
- **شهرياً**: اختبار استعادة كاملة للنظام
- **ربع سنوياً**: محاكاة سيناريوهات الطوارئ

### معايير النجاح
- **وقت الاستعادة**: أقل من 30 دقيقة للتراجع السريع
- **فقدان البيانات**: صفر أو أقل من ساعة واحدة من البيانات
- **توفر النظام**: 99.9% أو أكثر

## التحديثات والصيانة

### مراجعة الخطة
- **شهرياً**: مراجعة وتحديث الإجراءات
- **بعد كل حادث**: تحليل وتحسين الخطة
- **سنوياً**: مراجعة شاملة للخطة

### تدريب الفريق
- **تدريب أولي** لجميع أعضاء الفريق
- **تدريب دوري** كل 6 أشهر
- **محاكاة طوارئ** كل 3 أشهر
