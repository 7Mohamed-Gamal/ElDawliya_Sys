# دليل المدير - نظام الدولية

## نظرة عامة

هذا الدليل مخصص لمديري النظام والمسؤولين عن إدارة وصيانة نظام الدولية. يغطي جميع جوانب الإدارة من إعداد المستخدمين إلى مراقبة الأداء وحل المشاكل.

## 🔐 إدارة المستخدمين والصلاحيات

### إضافة مستخدم جديد

1. **تسجيل الدخول كمدير نظام**
2. **الإدارة** → **المستخدمين** → **إضافة مستخدم**
3. **املأ البيانات الأساسية**:
   ```
   اسم المستخدم: ahmed.mohamed
   البريد الإلكتروني: ahmed@company.com
   الاسم الأول: أحمد
   الاسم الأخير: محمد
   كلمة المرور المؤقتة: TempPass123!
   ```
4. **تحديد الصلاحيات**:
   - اختر الدور المناسب (موظف، مدير، مدير نظام)
   - حدد الوحدات المسموح بالوصول إليها
   - عين الصلاحيات المحددة لكل وحدة

### إدارة الأدوار والصلاحيات

#### الأدوار الافتراضية

| الدور | الوصف | الصلاحيات الأساسية |
|-------|--------|-------------------|
| **مدير النظام** | إدارة كاملة للنظام | جميع الصلاحيات |
| **مدير الموارد البشرية** | إدارة شؤون الموظفين | HR: إدارة كاملة، التقارير |
| **مدير المخزون** | إدارة المخزون والمشتريات | المخزون: إدارة كاملة، المشتريات |
| **مدير المشاريع** | إدارة المشاريع والمهام | المشاريع: إدارة كاملة |
| **موظف** | استخدام أساسي | عرض البيانات الشخصية، المهام |

#### إنشاء دور مخصص

1. **الإدارة** → **الأدوار** → **دور جديد**
2. **تحديد اسم الدور ووصفه**
3. **اختيار الصلاحيات**:
   - **عرض**: رؤية البيانات
   - **إضافة**: إنشاء سجلات جديدة
   - **تعديل**: تحديث البيانات الموجودة
   - **حذف**: إزالة السجلات
   - **موافقة**: اعتماد العمليات
   - **تصدير**: تصدير البيانات
   - **إدارة**: صلاحيات إدارية كاملة

### إعادة تعيين كلمة المرور

1. **البحث عن المستخدم** في قائمة المستخدمين
2. **اضغط "إعادة تعيين كلمة المرور"**
3. **اختر طريقة الإعادة**:
   - إنشاء كلمة مرور مؤقتة
   - إرسال رابط إعادة التعيين بالبريد الإلكتروني
4. **إشعار المستخدم** بالكلمة الجديدة

### إيقاف/تفعيل المستخدمين

- **الإيقاف المؤقت**: للموظفين في إجازة طويلة
- **الإيقاف الدائم**: للموظفين المنتهية خدمتهم
- **التفعيل**: لإعادة تشغيل الحسابات المتوقفة

## 📊 مراقبة النظام والأداء

### لوحة مراقبة المدير

تعرض المعلومات التالية:
- **المستخدمين النشطين**: عدد المستخدمين المتصلين حالياً
- **استخدام النظام**: الصفحات الأكثر زيارة
- **الأداء**: أوقات الاستجابة وسرعة التحميل
- **الأخطاء**: سجل الأخطاء والمشاكل التقنية
- **التخزين**: استخدام مساحة قاعدة البيانات والملفات

### تقارير الاستخدام

#### تقرير النشاط اليومي
```sql
-- مثال على استعلام تقرير النشاط
SELECT 
    u.username,
    COUNT(al.id) as login_count,
    MAX(al.login_time) as last_login,
    COUNT(DISTINCT al.ip_address) as unique_ips
FROM users u
LEFT JOIN audit_logs al ON u.id = al.user_id
WHERE DATE(al.created_at) = CURDATE()
GROUP BY u.id
ORDER BY login_count DESC;
```

#### تقرير استخدام الوحدات
- **الموارد البشرية**: عدد العمليات، المستخدمين النشطين
- **المخزون**: حركات المخزون، المنتجات المضافة
- **المشاريع**: المشاريع الجديدة، المهام المكتملة
- **المشتريات**: أوامر الشراء، قيمة المشتريات

### مراقبة الأداء

#### مؤشرات الأداء الرئيسية (KPIs)

| المؤشر | الهدف | الحالي | الحالة |
|---------|--------|--------|---------|
| وقت الاستجابة | < 2 ثانية | 1.5 ثانية | ✅ جيد |
| معدل الأخطاء | < 1% | 0.3% | ✅ ممتاز |
| وقت التشغيل | > 99% | 99.8% | ✅ ممتاز |
| استخدام الذاكرة | < 80% | 65% | ✅ جيد |
| استخدام المعالج | < 70% | 45% | ✅ ممتاز |

#### تنبيهات الأداء

إعداد تنبيهات تلقائية عند:
- وقت الاستجابة > 5 ثواني
- استخدام الذاكرة > 90%
- استخدام المعالج > 85%
- مساحة القرص الصلب < 10%
- فشل النسخ الاحتياطي

## 💾 إدارة النسخ الاحتياطية

### استراتيجية النسخ الاحتياطية

#### النسخ اليومية
- **قاعدة البيانات**: نسخة كاملة يومياً في 2:00 ص
- **الملفات**: نسخة تزايدية للملفات المتغيرة
- **الإعدادات**: نسخة من ملفات التكوين

#### النسخ الأسبوعية
- **نسخة كاملة**: جميع البيانات والملفات
- **اختبار الاستعادة**: التأكد من سلامة النسخ
- **أرشفة النسخ القديمة**: نقل النسخ القديمة للتخزين طويل المدى

### سكريبت النسخ الاحتياطي

```bash
#!/bin/bash
# backup_script.sh

# متغيرات التكوين
DB_NAME="eldawliya_db"
DB_USER="backup_user"
BACKUP_DIR="/backups/$(date +%Y%m%d)"
RETENTION_DAYS=30

# إنشاء مجلد النسخة الاحتياطية
mkdir -p $BACKUP_DIR

# نسخ قاعدة البيانات
mysqldump -u $DB_USER -p $DB_NAME > $BACKUP_DIR/database.sql

# نسخ الملفات
tar -czf $BACKUP_DIR/media_files.tar.gz /var/www/eldawliya/media/
tar -czf $BACKUP_DIR/static_files.tar.gz /var/www/eldawliya/static/

# نسخ ملفات التكوين
cp -r /etc/eldawliya/ $BACKUP_DIR/config/

# حذف النسخ القديمة
find /backups/ -type d -mtime +$RETENTION_DAYS -exec rm -rf {} \;

# إرسال تقرير النسخ الاحتياطي
echo "تم إنشاء النسخة الاحتياطية بنجاح في $BACKUP_DIR" | mail -s "تقرير النسخ الاحتياطي" admin@company.com
```

### استعادة النسخ الاحتياطية

#### استعادة قاعدة البيانات
```bash
# إيقاف الخدمات
sudo systemctl stop eldawliya-web

# استعادة قاعدة البيانات
mysql -u root -p eldawliya_db < /backups/20241201/database.sql

# إعادة تشغيل الخدمات
sudo systemctl start eldawliya-web
```

#### استعادة الملفات
```bash
# استعادة ملفات الوسائط
tar -xzf /backups/20241201/media_files.tar.gz -C /

# استعادة الملفات الثابتة
tar -xzf /backups/20241201/static_files.tar.gz -C /

# تعيين الصلاحيات
chown -R www-data:www-data /var/www/eldawliya/
```

## 📈 التقارير الإدارية

### تقارير الاستخدام

#### تقرير المستخدمين النشطين
- عدد المستخدمين المسجلين
- المستخدمين النشطين يومياً/أسبوعياً/شهرياً
- أوقات الذروة في الاستخدام
- الوحدات الأكثر استخداماً

#### تقرير الأداء
- أوقات الاستجابة للصفحات المختلفة
- معدل الأخطاء والمشاكل التقنية
- استخدام موارد الخادم
- إحصائيات قاعدة البيانات

### تقارير الأمان

#### تقرير محاولات تسجيل الدخول
```sql
SELECT 
    DATE(created_at) as date,
    COUNT(*) as total_attempts,
    SUM(CASE WHEN success = 1 THEN 1 ELSE 0 END) as successful,
    SUM(CASE WHEN success = 0 THEN 1 ELSE 0 END) as failed,
    COUNT(DISTINCT ip_address) as unique_ips
FROM login_attempts 
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY DATE(created_at)
ORDER BY date DESC;
```

#### تقرير الصلاحيات
- المستخدمين وأدوارهم
- الصلاحيات المعينة لكل مستخدم
- التغييرات الأخيرة في الصلاحيات
- المستخدمين غير النشطين

## 🛠️ استكشاف الأخطاء وإصلاحها

### المشاكل الشائعة وحلولها

#### مشكلة: النظام بطيء
**الأسباب المحتملة:**
- استعلامات قاعدة البيانات غير محسنة
- نقص في ذاكرة الخادم
- عدد كبير من المستخدمين المتزامنين

**الحلول:**
1. **تحسين الاستعلامات**:
   ```sql
   -- إضافة فهارس للجداول الكبيرة
   CREATE INDEX idx_employee_department ON employees(department_id);
   CREATE INDEX idx_attendance_date ON attendance(attendance_date);
   ```

2. **زيادة ذاكرة الخادم**:
   ```bash
   # تحديث إعدادات MySQL
   innodb_buffer_pool_size = 2G
   query_cache_size = 256M
   ```

3. **تفعيل التخزين المؤقت**:
   ```python
   # في settings.py
   CACHES = {
       'default': {
           'BACKEND': 'django_redis.cache.RedisCache',
           'LOCATION': 'redis://127.0.0.1:6379/1',
           'OPTIONS': {
               'CLIENT_CLASS': 'django_redis.client.DefaultClient',
           }
       }
   }
   ```

#### مشكلة: أخطاء في قاعدة البيانات
**الأسباب المحتملة:**
- فساد في البيانات
- مساحة القرص الصلب ممتلئة
- مشاكل في الاتصال

**الحلول:**
1. **فحص سلامة قاعدة البيانات**:
   ```sql
   CHECK TABLE employees;
   REPAIR TABLE employees;
   ```

2. **تنظيف مساحة القرص**:
   ```bash
   # حذف ملفات السجلات القديمة
   find /var/log/ -name "*.log" -mtime +30 -delete
   
   # تنظيف ملفات MySQL المؤقتة
   mysql -e "PURGE BINARY LOGS BEFORE DATE_SUB(NOW(), INTERVAL 7 DAY);"
   ```

#### مشكلة: المستخدمون لا يستطيعون تسجيل الدخول
**التحقق من:**
1. **حالة الخدمات**:
   ```bash
   sudo systemctl status eldawliya-web
   sudo systemctl status mysql
   sudo systemctl status redis
   ```

2. **سجلات الأخطاء**:
   ```bash
   tail -f /var/log/eldawliya/error.log
   tail -f /var/log/mysql/error.log
   ```

3. **اتصال قاعدة البيانات**:
   ```python
   # اختبار الاتصال
   python manage.py dbshell
   ```
